## 2. CONTEXT AND MOTIVATION

Modern experimental research suffers from persistent
issues of poor traceability, loss of contextual
knowledge, and limited reproducibility. Traditional
scientific training prioritises execution of experiments
over rigorous documentation, contributing directly to
the reproducibility crisis. This TFG addresses that gap
by developing a modern, reusable documentation
framework, aligned with FAIR principles, Open Science
practices, and real research-centre constraints.

---

## 3. OBJECTIVES

### 3.1 Primary Objective

To design, implement, and validate a modern
documentation framework for experimental research
projects, improving traceability, quality, and
reproducibility.

### 3.2 Secondary Objectives

- Analyse the state of the art in scientific
  documentation tools and methodologies.
- Select and justify a set of open or widely
  accessible digital tools.
- Apply the framework to a real but abstracted
  pilot case.
- Produce a concise and practical Guide of Best
  Practices for future students and researchers.

---

## 4. SCOPE DEFINITION

### 4.1 In Scope

- Knowledge management methodology
- Documentation structures and templates
- Metadata, traceability, and versioning concepts
- Conceptual use of RACO project objectives
- Abstracted pilot validation at IMB-CNM

### 4.2 Out of Scope

- BACO production workflows
- Fabrication RUNs, recipes, or parameters
- Operational YAML or execution data
- Any confidential CSIC technical information

---

## 5. METHODOLOGY AND WORK PLAN

### Phase I: Research and Analysis (15%)

- Literature review (FAIR, Open Science,
  reproducibility)
- Analysis of ELNs, Git-based workflows,
  executable documentation
- Needs analysis survey

### Phase II: Framework Design (25%)

- Tool selection and justification
- Definition of directory structure
- Creation of reusable documentation templates

### Phase III: Implementation and Validation (40%)

- Application to a pilot project at IMB-CNM
- Active documentation (not experimental execution)
- Iterative refinement based on real use

### Phase IV: Synthesis and Writing (20%)

- Writing of the TFG memory
- Production of an independent Guide of Best
  Practices
